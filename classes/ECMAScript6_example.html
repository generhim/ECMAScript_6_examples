ECMAScript 6 Example: Introduces the "class" keyword moving JavaScript closer to other C-based languages.

  1. ECMAScript 1 - 5 does not have classes.
  2. ECMAScript 6 - Version 6 DOES use classes. We are no longer bound by functions and prototypes.
  3. In the case of version 6, we are creating a new class, "ClassName" {} which has methods that do NOT require the function keyword in front of them, and do NOT need to reference an object's prototype.
  4. Instead the var MyNewPerson = new MyPerson("NameGoesHere") and MyNewPerson.whatIsMyName() are equivalent to the function-based logic created with earlier versions of ECMAScript 5, etc.
